# 2026 企业级 AI 桌面终端交互范式调研报告

## 1. 调研背景与核心洞察
随着 Agentic Workflow（智能体工作流）在企业场景的落地，业界领先的 AI 产品的交互设计已经全面摒弃了早期的“Chatbot（聊天机器人）”和“空白无界画布（Infinite Canvas）”。
企业场景（尤其是产研和职能）的核心诉求是：**效率、可控性、确定性以及极低的认知负荷**。无界画布虽然自由，但带来了过高的空间认知成本，不符合企业 SOP（标准作业程序）的要求。

当前主流企业级 AI 的交互范式已收敛为以下三大阵营：

## 2. 业界三大主流交互范式盘点

### 范式一：“全知输入与卡片式结果” (Omni-Search & Action Cards)
*   **代表产品**：Glean (企业知识搜索独角兽), Raycast Enterprise
*   **交互逻辑**：
    *   **没有主界面**：常态下只是一个系统级的居中搜索框（类似 macOS Spotlight）。
    *   **指令即应用**：用户输入自然语言（如“查阅昨日贝易转客诉”），系统后台连接内部知识库、Jira、Salesforce 等。
    *   **卡片化呈现**：不生成长文本，而是直接在输入框下方弹出一个高度结构化的「数据卡片」或「操作卡片」，用完即走。
*   **企业价值**：极致的“快”，适合高频、轻量的查询与审批动作。

### 范式二：“上下文悬浮侧车” (The Copilot Sidecar / Screen-Aware Agent)
*   **代表产品**：Adept AI (ACT-1), Microsoft Copilot for Windows
*   **交互逻辑**：
    *   **吸附于现有工作流**：AI 的 UI 悬浮在屏幕右侧或作为一个极窄的侧边栏。它**不创造新的工作空间**，而是基于用户当前正在使用的界面（无论是 Chrome 里的老旧 ERP，还是本地的 Excel）进行操作。
    *   **代替点击（RPA 2.0）**：UI 重点展示“AI 正在替你做的事”。用户发出指令后，AI 在侧边栏展示思考步骤，并在主屏幕的业务系统中直接模拟操作。
*   **企业价值**：无需改造企业现有系统（Legacy Systems），完美契合复杂表单填写、跨系统数据搬运等标准工作流。

### 范式三：“双栏制品工作台” (Pane-based Artifact Workspace)
*   **代表产品**：Cursor (AI IDE), Harvey (法律 AI 助手), Anthropic Claude Artifacts
*   **交互逻辑**：
    *   **结构化的区域划分**：彻底摒弃“无界自由”，采用极其克制的“双栏/三栏”窗格。
    *   **左侧（成果区/制品区）**：展示业务实体（一整段代码、一份数据报表、一份待确认的配置单）。
    *   **右侧（Agent 轨迹区）**：展示数字员工的思考链条、调用的工具记录（Sensors/Actuators）。
    *   **差异对比（Diff & Review）**：Agent 提出的修改或执行动作，会以可视化的 Diff 形式覆盖在左侧，强制要求用户点击“Accept（接受）”或“Reject（驳回）”。
*   **企业价值**：提供极强的**安全感与审计追踪（Auditability）**。对于产研和职能人员处理复杂数据或执行线上配置修改，这是唯一能保证确定性的交互方式。

### 特别案例补充调研：Claude Co-work 与 GPT Workspace (Canvas)
除了上述三大通用范式，近期业界最引人注目的两款重磅产品提供了更具体的交互启示：

*   **Claude Co-work (Anthropic 的桌面级 Agentic UI)**：
    *   **设计理念**：不把 AI 当作聊天对象，而是“全自动的数字同事”。底层架构与纯终端环境下的 Claude Code 一致，但专门为产研/职能人员包了一层极其直觉的 GUI。
    *   **交互亮点 - “后台托管与沙箱签批”**：它的 UI 极力弱化聊天框，强化**“任务追踪条”**。当让它处理跨系统的数据整理或复杂分析时，界面会收缩为一个极简的进度监视器，安静地在后台沙箱运行。只有当涉及敏感修改或执行器越界时，主屏才会弹出一个强制的 **“Action Sign-off (动作签批单)”**，完美融合了“无感操作”与“硬性风控”。

*   **GPT Workspace & Canvas (OpenAI 的企业级协同界面)**：
    *   **设计理念**：OpenAI 面向企业长文本、复杂数据协作推出的重磅形态。
    *   **交互亮点 - “基于选区的局部修改 (Inline Editing)”**：在 Canvas 模式下，左侧是占据大屏的“业务制品（数据报表/文档）”，右侧是极窄的 AI 侧边栏。这里的核心创新是：**AI 退居二线，业务数据才是主角**。用户可以像用鼠标一样圈选左侧图表中的某一行，右侧的 AI 侧边栏会自动吸附上下文并进行“局部就地修改”。这种“高亮哪里改哪里”的隐喻，极大地降低了用户让 AI 全盘重写带来的不确定性恐慌。

*   **Workspace Agents (OpenAI 2026 最新发布 - 真正的工作空间智能体)**：
    *   **设计理念**：将智能体从“聊天插件”进化为“云端自治同事”。
    *   **交互亮点 - “异步代理与多端渗透”**：
        1.  **异步脱机执行 (Background Autonomy)**：用户下达任务后可以关机或离线，智能体在云端持续执行（数小时甚至更久），完成后通过 Slack 或系统通知反馈结果。这彻底改变了“对话即执行”的实时交互模式。
        2.  **双栏构建器 (Dual-Pane Builder)**：左侧自然语言交互，右侧实时生成结构化的智能体定义（Identity/Tools/Knowledge），通过视觉上的“实时反馈”降低用户的配置焦虑。
        3.  **强制性动作审批 (Mandatory Sign-off)**：对于发送邮件、修改财务数据、变更日程等敏感操作，系统强制弹出简洁的审批卡片。
    *   **企业治理**：提供全局的审计日志和“行为看板”，让管理员能够清晰追踪每个智能体调用了哪些内部数据、执行了哪些系统操作。

## 3. 核心演进趋势 (Key Trends)

1.  **From Chat to Artifacts（从“对话”到“制品”）**
    企业用户不需要一个“会聊天的系统”，他们需要系统交付“工作制品（Artifacts）”（如：图表、看板、执行计划单）。现代 UI 会自动折叠啰嗦的对话，将生成的制品最大化铺满屏幕核心区域。
2.  **Human-in-the-loop (HITL) 的具象化呈现**
    企业 AI 设计最重要的一环是“人机互信”。在调用 Actuator（执行器）去修改业务系统时，UI 必须强制提供“预执行预览（Dry-run）”和“签批阻断（Sign-off）”机制。
3.  **UI 状态机的收敛（Focus Mode）**
    企业级终端在处理单个复杂任务时，倾向于进入“专注模式”，屏蔽外部干扰，只显示当前任务所涉及的 Sensor 来源和预期 Actuator 动作，直到流程闭环。

## 4. 对 Kevin 1.5 的修正与启示
基于以上调研，放弃“无界画布”是非常正确的业务直觉。对于 Kevin 的 PC 端设计，我建议融合 **“范式一”** 与 **“范式三”**：

*   **轻量态（Glean 模式）**：日常完全隐形，通过快捷键唤出**全能输入框**。处理查数据、找文档等轻任务时，直接在输入框下渲染卡片，用完即走。
*   **重度态（Cursor 制品工作台模式）**：当涉及“贝易转复杂数据流转及营销策略下发”这种长链路任务时，展开为**严谨的双栏工作台**。
    *   主视区渲染数据报表和“待签批的策略修改单”（Artifacts）。
    *   侧边栏展示 Kevin 的数据获取逻辑与调用轨迹，让用户可随时审查，提供企业级的安全感。
