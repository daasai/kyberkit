## 1. 目标与原则

### 1.1 程序目标（团队共识）

- **交付节奏**：不设硬性上线日期；以走查条目 **`fixed` → `verified`**、三场专题决议落地、第二轮 UAT 复测通过为收口（见 [`uat_walkthrough_live_log_2026-05-07.md`](./uat_walkthrough_live_log_2026-05-07.md)）。
- **体验与可靠性双极致**：**用户体验**与**系统可靠性**同等优先级，不设「先凑合再 polish」的默认路径；可接受的妥协须经书面记账（范围说明或后续版本）。

### 1.2 原则表

| 原则 | 说明 |
|------|------|
| **PRD 契约优先** | 以 `kevin-v1.5-prd-rev2.md` In v1.5 / §7 / §11 为验收锚点；缺陷项优先于纯体验增补。 |
| **先止血、再重构** | P0 阻断与 **Space / 会话语义**（009）必须先正确，否则文档库、历史会话、右栏全部可能连锁错位。 |
| **与退化修复套餐对齐** | 代码侧已识别 **prop 链断裂**（搜索、Skill Store、自动化无响应等），应先执行「外科手术」规格，再叠加走查中的 **语义层与 IA 升级**。 |
| **设计专题单列** | Header、文档库（Obsidian 向）、右栏（快速启动 + 过程追踪）需单独评审，避免在未定稿前大面积返工。 |
| **UX 极致** | 信息架构清晰、状态可理解、空态/加载/失败均有明确文案与下一步；关键路径减少认知跳转；灵动岛、侧栏、三栏职责与 PRD 语义一致；专题会议产出视觉与交互规范后再批量实现。 |
| **可靠性极致** | Space / 会话 / Sidecar 边界可证明正确（回归与集成测试）；失败可观测（用户可见原因 + 日志可追踪）；竞态与离线降级有定义；附件、文档库树、连接器与大目录场景经过性能与边界用例。 |

## 2. 工作流（Workstreams）与 UAT 映射

### WS-0 — 接线退化修复（前置）

- **来源**：`2026-05-07-kevin15-regression-fix-and-header-design.md`（R-01～R-07、W-01～W-05）。
- **直接缓解走查**：003（搜索无反应）、004/007（左栏与路由打不开）、009 的部分诱因（SessionContext `spaceId` 缺失）、灵动岛 idle（W-02/R-06）、顶部按钮哑火（W-03）。
- **交付**：恢复 `AppShell` 路由与 props、`SessionProvider` 中 `spaceId`、Sidecar 查询带 Space 等。

### WS-A — Space / 会话模型（架构语义）

| UAT | 要点 |
|-----|------|
| **009** | 底部控件必须切换 **Space（`space_id`）**，不得与会话列表混用。 |
| **008 / 010** | 历史会话：PRD 差距清单；点击 **in-place** 更新中栏制品 + 右栏对话流，禁止默认新开窗口。 |
| **依赖**：WS-0 中 Session / Sidecar 空间作用域修复完成后，009、010 才能稳定验收。 |

### WS-B — 左侧栏：搜索、分组、文档库、连接器

| UAT | 类型 | 要点 |
|-----|------|------|
| **003** | 缺陷 | 搜索点击有响应（检索面板或路由至 CenterPanel 搜索视图，与 PRD §7 一致）。 |
| **005** | 增补 | 「新建会话 + 历史会话」同区块分组，与文档库/连接器 **分隔线** 区隔。 |
| **006** | 缺陷 + 升级 | **A** 映射正确性（当前 Space → workspace data）；**B** 树形 IA（Obsidian 参考）与懒加载。 |
| **007** | 缺陷（待验真） | 最小可用：配置入口 + 状态/空态；或明确「未开放」而非静态占位。 |

### WS-C — 中栏：制品主阵地

| UAT | 要点 |
|-----|------|
| **011** | Artifact Toolbar：Export / Share / 编辑·预览 — 不可用则 **降级展示**（禁用 + 原因）。 |
| **013-②** | 「已有 x 项产物」→ **可点击制品列表**或跳转 `artifact-primary-view`。 |

### WS-D — 右栏：输入能力 + 过程追踪 + 快速启动

| UAT | 要点 |
|-----|------|
| **012** | Input Toolbar（📎、`@`、`/`）与 Skill 列表数据源统一（文档库 / Sensor / 已安装 Skill）。 |
| **013** | 轨迹与 **chat stream 去重**；轨迹保留则只做 **结构化摘要**，不做第二套流式正文。 |
| **014** | 快速启动与 013 **捆绑重设计**（专题 TODO）。 |
| **015** | 粘贴文件 + 拖放 → 归入附件管线（与 012 同一里程碑）。 |

### WS-E — Skill Store

| UAT | 要点 |
|-----|------|
| **004** | 「新建私有 Skill」：**补齐最小流程** 或 **移除/置灰 CTA**（与 Forge / §9 叙事一致）。 |

### WS-F — 顶部：灵动岛 + Header IA

| UAT | 要点 |
|-----|------|
| **001** | 去输入框化、拟人化状态文案（可配置语气库）。 |
| **002** | Search / Skills / 自动化迁 Header、竖线分组 — **依赖 Header 专题会议定稿**（与走查 TODO 一致）。 |

## 3. 分阶段路线图（建议）

### 阶段 0 — 立即（1～3 天量级，依人力）

1. 落地 **WS-0** 退化修复规格（与现有 PR 分支对齐）。
2. 并行：**004** 文案/CTA 止血（最快降低误导）。
3. **007** 技术验真（三选一：未实现 / 未接线 / 仅缺状态）。

### 阶段 1 — P0 主路径（依赖阶段 0）

顺序建议：

1. **009** Space 切换语义 → **008/010** 历史会话 in-place + 差距清单闭合。
2. **003** 搜索可用；**006-A** 文档库目录映射正确（先于大规模 UI 换肤）。
3. **011 / 012 / 013** 中栏制品条 + 右栏输入 + 轨迹/产物导航（可按子 PR 拆分）。

### 阶段 2 — 体验收敛与设计专题产出

1. **文档库专题会议**：006-B（Obsidian 向）+ 底层树模型 / 性能 / 错误态（走查 TODO）。
2. **右栏专题会议**：013 + 014（快速启动 + 过程追踪统一 IA）。
3. **Header 专题会议**：001 + 002 + 灵动岛与 Logo 关系。

### 阶段 3 — 增补项

- **005** 侧栏分组调整。
- **015** 拖放/粘贴（在 012 附件管线稳定后验收）。
