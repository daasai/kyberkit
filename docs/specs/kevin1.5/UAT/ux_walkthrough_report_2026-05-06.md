# Kevin v1.5 UX 走查报告（基于当前界面截图）

- 日期：2026-05-06
- 走查角色：UX 负责人
- 评审依据：`docs/specs/kevin1.5/kevin-v1.5-prd-rev2.md`
- 评审对象：当前 Kevin 主界面截图（重度态）
- 评审方式：静态界面对照 PRD 条款走查（非交互录屏）

---

## 1. 结论概览

当前界面已具备 v1.5 三栏雏形（左导航、中制品区、右对话区），但与 PRD Rev2 仍存在多处关键偏差，主要集中在三类：

1. 核心信息架构偏差（Space 锚点缺失，底部被 Upgrade Plan 占位）
2. 任务状态可感知性不足（灵动岛四态不完整）
3. 产品心智风险（欢迎区内容暗示预装 Skill）

若不优先修正，将影响用户对 v1.5 的核心认知：Space 隔离、多窗口协同、异步任务与签批闭环。

---

## 2. 问题清单（按严重程度）

### P0 / High

#### 2.1 左下角 Space 锚点缺失，被 Upgrade Plan 占位
- 现状：左下为 `Upgrade Plan`，`Space / Vault` 位于左栏中上部。
- PRD 期望：Space 切换器作为左栏最底部固定锚点（Vault 隐喻）。
- 风险：用户从“工作空间导航”被引导到“商业升级入口”，破坏核心任务路径。
- 对照条款：§7.1E、§13。
- 建议：
  - 左下固定为 Space Switcher（当前 Space + 切换入口 + 新建 Space）。
  - 将 `Upgrade Plan` 移至设置或头像菜单，退出主任务动线。
  - 交互范式参考 Obsidian Vault Switcher。

#### 2.2 灵动岛未体现 v1.5 四态任务反馈
- 现状：顶部中间更像静态长条提示区，缺少进度环、待签批红色脉冲、完成瞬态摘要等状态语义。
- PRD 期望：空闲 / 执行中 / 待签批 / 完成瞬态 四态完整。
- 风险：异步任务与签批状态不可见，用户无法建立系统可控感。
- 对照条款：§7.2、§11.3、§10.2。
- 建议：
  - 空闲态显示可编辑会话标题。
  - 运行态显示任务名 + ETA + 进度环。
  - 待签批态显示红色脉冲与待处理数量。
  - 完成态展示 3 秒摘要后自动收起。

#### 2.3 欢迎区内容存在“预装 Skill”误导
- 现状：中栏欢迎区提供“站会数据 / 产品 Spec / RCA 分析”等建议入口。
- PRD 期望：v1.5 不预装任何 Skill，示例仅为契约蓝本。
- 风险：与产品范围冲突，造成预期管理失败。
- 对照条款：§9.0、§8.5。
- 建议：
  - 改为“如何开始”引导（@文件、附件、自然语言任务、Slash 调用已安装 Skill）。
  - 若展示样例，应明确“示例任务，不代表预装技能”。

### P1 / Medium

#### 2.4 Sensor 状态区信息不足，不满足最小可恢复闭环
- 现状：仅看到传感器列表，未见健康灯和恢复动作。
- PRD 期望：状态灯（绿/黄/红）+ 失败原因 + 最近成功时间 + 一键重连/重授权。
- 风险：故障不可诊断，可靠性心智下降。
- 对照条款：§7.1C、§15.1（Sensor 健康可见性 100%）。
- 建议：
  - 每个 Sensor 行增加状态灯和最近状态文案。
  - 点击展开卡片，展示失败原因与恢复按钮。

#### 2.5 历史会话区关键交互可发现性不足
- 现状：有标题+时间列表，但截图中未体现置顶/存档、6条折叠、待签批红点。
- PRD 期望：Codex 式行布局 + 悬浮操作 + 折叠策略 + 签批标记。
- 风险：历史管理效率低，签批提醒漏感知。
- 对照条款：§7.1D、§10.2。
- 建议：
  - 默认展示 6 条，补 `Show more`。
  - hover 明确显示置顶/存档图标。
  - 待签批会话行增加红色脉冲点。

#### 2.6 右侧面板结构偏离“对话与过程追踪器”定位
- 现状：右侧包含“对话/快速启动”标签与“制品”区域，结构语义混杂。
- PRD 期望：右侧以输入与过程追踪为主，中栏承载制品主视图。
- 风险：任务主副关系混乱，用户不清楚“在哪里看结果，在哪里下指令”。
- 对照条款：§7.3。
- 建议：
  - 右侧聚焦消息流 + 输入区 + 状态内联条目。
  - 制品主视图回归中栏，右侧仅保留制品跳转/引用。

#### 2.7 `Context & Sources` 区域信息架构不清，且未满足 PRD 的“文档库+连接器”双目标
- 现状：`Context & Sources` 以单一扁平区块混合展示“本地文件入口”和“MCP/Sensor 项”，用户难以区分“可引用上下文”与“系统健康状态”。
- PRD 期望：左栏应显式支持文档库双向联动（可浏览、可 `@` 引用）以及连接器状态区最小闭环（状态灯、失败原因、最近成功时间、重连/重授权）。
- 风险：
  - 用户在发起任务前无法快速建立“可用上下文”心智；
  - 连接器异常时缺少可诊断和恢复入口，影响可靠性感知；
  - 区块体量与层级会挤压历史会话区，削弱会话管理效率。
- 对照条款：§7.1B、§7.1C、§7.3（Input Toolbar 的 `@` 提及语义）。
- 建议（展开版）：
  - **区块拆分**：将 `Context & Sources` 拆分为 `文档库` 与 `连接器` 两个独立子区，并支持各自折叠/展开。
  - **文档库（类 Obsidian 目录）**：
    - 展示线上 Workspace 文件树（按目录层级浏览、展开、折叠）。
    - 文件/目录行提供快速动作：`@引用`、`复制路径`（可选）。
    - 保留“最近引用”小区块（3-5 条），降低重复检索成本。
    - 空态文案聚焦任务导向：`在文档库中选择文件，或通过 @ 快速引用到对话`。
  - **连接器（原 Sensors，中文化）**：
    - 区块标题统一改为 `连接器`，避免英文术语割裂。
    - 每项展示：状态灯（🟢/🟡/🔴）+ 连接器名称 + 最近成功时间。
    - 点击行展开详情：失败原因（人类可读）、影响范围、`重连/重授权` 按钮。
    - 异常连接器排序置顶，保证问题优先暴露。
  - **布局优先级**：
    - 文档库优先占用垂直空间（主任务入口）。
    - 连接器区默认折叠为摘要态（如“2 正常 / 1 异常”），展开后查看详情。
    - 始终保证“历史会话”区至少可见 6 条，符合 PRD 的会话管理要求。

### P2 / Low

#### 2.8 中右视觉权重失衡，主任务焦点不够清晰
- 现状：中栏留白较大，右栏容器与输入框存在感更高。
- PRD 期望：业务制品是主角，AI 交互是辅助手段。
- 风险：用户误判主工作区，降低复杂任务效率。
- 对照条款：§6.1、§7.3。
- 建议：
  - 增强中栏制品空态引导与默认结构骨架。
  - 降低右栏空态视觉噪声，突出“输入一次，结果在中栏”。

## 3. Space 区专项建议（按 Obsidian 参考图修正）

本节按你提供的 Obsidian 交互范式进行收敛，目标是让 Space 成为左侧栏稳定锚点，而非二级设置项。

### 3.1 位置与结构（固定底部锚点）

- 左侧栏最底部固定 `Space Switcher`，替换现有 `Upgrade Plan` 占位。
- 底部主按钮形态：`[Space 图标] [当前 Space 名称] [下拉箭头]`。
- 右侧放置两个轻量动作图标：
  - `?`：Space 使用帮助（可选）
  - `⚙`：Space 管理入口（重命名、归档、删除、权限）
- `Upgrade Plan` 从左栏主路径移除，迁移至“头像菜单”或“设置-账单”。

### 3.2 弹出菜单（下拉）规范

点击底部 Space 按钮后弹出菜单，结构参考 Obsidian Vault 列表：

1. **Space 列表区**（主区）
   - 每行：`Space 名称`（必显）
   - 可选副信息：最近访问时间 / 本地路径摘要（二选一，避免噪声）
   - 当前 Space：`✓` 标记 + 行高亮
   - 若 Space 存在 `awaiting-signoff`：行尾红点提示
2. **分隔线**
3. **管理操作区**
   - `管理 Space...`（统一进入完整管理页）
   - `新建 Space`（可放在管理页或菜单底部，二选一）

### 3.3 交互行为规范（必须与 PRD 一致）

- 切换 Space 的语义是“切换到独立窗口”：
  - 点击非当前 Space -> 打开目标 Space 新窗口并聚焦；
  - 当前窗口保持原 Space，不做“就地替换”。
- 点击当前 Space 行：
  - 若当前窗口已是该 Space，仅关闭菜单；
  - 若该 Space 已有窗口在前台，聚焦到该窗口。
- 键盘支持：
  - `Cmd+\`` 进行 Space 窗口循环（沿用系统行为）
  - 菜单支持上下键选择、回车确认。

### 3.4 文案与命名建议

- 左栏文案统一用 `Space`（英文）或 `工作空间`（中文），避免 `Vault` 与 `Space` 混用造成理解成本。
- 菜单项建议：
  - `管理 Space...`
  - `新建 Space`
  - `在新窗口打开`（如需显式二次动作时）

### 3.5 视觉与可用性细节

- Space 名称超长时单行省略，hover 展示全名。
- 列表最大高度建议 40-60vh，超出滚动，保持底部菜单不遮挡主工作区。
- 当前 Space 与普通项采用明显对比（背景色+勾选），避免“看不出当前所在空间”。
- 红点告警只用于待签批，不复用为一般通知，防止语义污染。
