## 3. E2E 主链路证据

`src-sidecar/e2e_mvp_rc.test.ts`：单个 spec 串联 6 步 MVP-RC 核心叙事，每一步均断言。

1. **Forge P0**：`suggestForgeDraft({ message: '以后都这样：把会议纪要导出为 SKILL' })` 返回 `trigger='explicit'` → `acceptForgeDraft` 落盘到 `${spaceSkillsDir}/meeting-export/SKILL.md`。
2. **Scanner**：`scanSkillsForSpace(spaceId)` 命中 `meeting-export`，`scope='space'`。
3. **LearningLoop L1**：`appendStyleNote` → `loadStyleNotes` 读出 → `composeSkillBody` 在 L2 体内拼入。
4. **Cron 桥**：写入带 `kevin.cron: "0 9 * * 1-5"` 的 `standup` skill → `syncCronForSpace` 注册 1 个 job → `_tickNowForTest` 触发 → `TaskManager.list(spaceId)` 收到 `trigger_kind='cron'` 任务。
5. **Sign-off + 审计**：`previewFeishuDocDiff` → `createSignoffTask` → `runFeishuDocWriteMock` 生成本地 `mock-doc-*.md` → `resolveSignoff(true)` → `appendAudit('approved')`；审计 jsonl 同时含 `pending` 与 `approved` 两行。
6. **通知聚合**：`aggregateNotifications` 根据 1h 窗 + 阈值正确产出至少一组卡片（`aggregate` 或 `completed`）。

## 5. 已知延后项（与计划 §3 一致）

> 与 v1.5 全量 PRD 相比，下列特性在 MVP-RC 主动延后；与上一轮 UAT 报告记录一致。

- Skill Forge **P1 自动模式检测**（≥3 次相似对话且 7 天内）。
- LearningLoop **Layer 2/3**（Skill 自演进、跨 Space/User 共享）。
- HITL **`high` 风险**（外发邮件 / 公开发布）走 v2.0 多人 + 二次输入。
- 灵动岛 “主对话内嵌入” 的下一步迭代（重命名 / 折叠摘要）。
- `crontab.json` 落盘 + 重启重放（v1.5 仅内存调度，重启会丢未触发任务，已在 `CronScheduler.ts` JSDoc 中标注）。
- 远端轨迹回传 / 网络化 Skill 共享。
