### UAT-20260507-001

- 模块：顶部灵动岛（Dynamic Island）
- 场景：任务执行态文案与视觉呈现
- 问题描述：当前灵动岛样式过于接近输入框（边框/底色感较强），用户第一感知偏“可编辑控件”而非“系统状态提示”；执行态英文文案 `Processing request` 缺乏拟人化与安抚感。
- 期望结果：改为无边框、弱底色（或纯文字）+ 标签化状态；文案采用用户语义表达（如“XX 请求正在处理，您可以喝杯咖啡休息一下 ☕️”），降低等待焦虑。
- 实际结果：现有形态偏表单控件风格，文案偏系统术语风格。
- PRD 对照：`kevin-v1.5-prd-rev2.md` 明确灵动岛为 v1.5 P0 一等公民并要求状态反馈清晰（§11.3）；但未强制指定“无边框+拟人化”视觉与文案风格。
- UX 评估：可发现性=中（能看到但角色认知偏差）；可理解性=中（术语化表达）；操作负担=低（无需额外操作）；一致性=中（与“iPhone 范式”目标存在语义与视觉距离）。
- 判定分类：`v1.5-enhancement`
- 严重级别：S2（体验重要，非功能阻断）
- 优先级：P1（建议纳入 v1.5 体验收敛）
- 状态：`triaged`
- 备注：建议拆分为两个可并行子项：A) 视觉去输入框化；B) 执行态文案本地化与拟人化策略（含可配置语气库）。

### UAT-20260507-002

- 模块：App Header 顶部信息架构（Logo / 灵动岛 / 顶部入口）
- 场景：全局导航入口布局与操作可达性

- 期望结果：顶部形成“左品牌+状态、中/右功能入口、最右系统操作”的清晰结构，减少跨区域视线跳转与点击路径。
- 实际结果：当前入口分布在 Sidebar 与 Header 两处，功能聚合度不足，顶部区块分组边界不明显。
- PRD 对照：`kevin-v1.5-prd-rev2.md` 要求 v1.5 交付左/中/右三栏 + 顶部灵动岛，但未硬性规定 `Search/Skills/自动化` 必须位于 Sidebar 或 Header（§7、§11.3、In v1.5 条款）。
- UX 评估：可发现性=中（入口分散导致首次定位成本上升）；可理解性=中高（分组后可提升）；操作负担=中（跨栏位操作路径偏长）；一致性=中（Header 作为全局控制中枢的语义尚未完全建立）。
- 判定分类：`v1.5-enhancement`
- 严重级别：S2（结构优化价值高，非功能阻断）

- 严重级别：S1（关键能力预期误导）
- 优先级：P0（建议尽快修复“功能补齐或文案降级”其一）
- 状态：`verified`
- 备注：本次采用止血路径 A：保留入口可发现性，但禁用 `+ 新建私有 Skill` 并通过 `title` / 说明文案标注 Forge 落盘前置条件。回归测试：`SkillStore.test.tsx` 覆盖禁用状态与说明文案。

### UAT-20260507-005

- 模块：Sidebar 信息架构（会话分组）
- 场景：“新建会话/历史会话”与“文档库/连接器”的布局关系
- 问题描述：`新建会话` 与 `历史会话` 在用户心智中属于同一任务域（会话管理），但当前被 `文档库` 与 `连接器` 分隔，可能造成功能归属混淆与操作路径中断。
- 期望结果：会话相关入口在视觉与结构上保持连续分组；跨域模块（文档库、连接器）与会话模块具备明确层级与边界。
- 实际结果：会话功能被跨域模块打断，用户可能误判导航主线。
- PRD 对照：`kevin-v1.5-prd-rev2.md` 要求三栏信息架构清晰与复杂度收敛（§6、§7），但未规定 Sidebar 的精确顺序；因此该问题属于 IA 收敛质量而非硬性条款违约。
- UX 评估：可发现性=中（入口都可见）；可理解性=中低（分组语义不连续）；操作负担=中（切换历史会话时存在视线跳跃）；一致性=中低（同域功能被拆散）。
- 判定分类：`v1.5-enhancement`
- 严重级别：S2（认知效率问题，非功能阻断）
- 优先级：P1（建议在 v1.5 体验收敛阶段完成）
- 状态：`triaged`
- 备注：建议“分组整合而非功能合并”——保留独立入口，但将 `新建会话 + 历史会话` 置于同一会话区块，并通过分隔线与 `文档库/连接器` 区块隔离。

- 场景：点击历史会话列表中的某一条
- 问题描述：当前点击历史会话会 **打开新窗口**（或等效的多窗口跳转），用户期望在同一主窗口内，于 **中栏** 加载该会话相关 **制品**，于 **右栏** 加载 **对话流**（及过程追踪等 PRD 约定内容），保持三栏 IA 不被打断。
- 期望结果：历史会话点击 = 应用内路由：切换当前活动会话上下文，CenterPanel / RightPanel 刷新为该会话的数据；不默认 spawn 新 OS/Tauri 窗口（除非 PRD 另有显式“在新窗口打开会话”的二级动作）。
- 实际结果：行为偏离上述模型，窗口增殖、上下文割裂。

- 备注：已与 `open_and_focus_space_window` 等多窗口能力区分：Space 级新窗口为显式 Space 动作；会话级点击仅 `setActiveSessionId` in-place 切换。回归测试：`npm run test:vitest --prefix app -- --run src/components/layout/LeftSidebar.test.tsx`（3 tests passed）。

- 备注：修复后可补充验收——「仅有一处展示原始模型流式正文」「点击产物提示可在中栏聚焦对应制品」。
