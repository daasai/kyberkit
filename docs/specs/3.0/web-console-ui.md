# Web Console MVP — UI 设计 (3.0 P1-C)

状态: Draft  
范围: P1-C — Web Console 用户界面层（不含后端 API、不含部署）  
目标读者: 产品 / 前端实现者

---

## 0. 设计参照与产品边界

### 0.1 参照来源

布局与信息层级参照 **Codex 应用**在非编程场景下的极简结构（见产品内截图）：**窄侧栏**承载高频动作与历史；**主区**为对话流；**第三栏**承载网页等制品预览。能力叙事上仍可对齐 [Workspace Agent 公告](https://openai.com/zh-Hans-CN/index/introducing-workspace-agents-in-chatgpt/) 中的「进度、文件、工具、下一步」与权限心智。

KyberKit Web Console **不是** ChatGPT/Codex 的复刻：无云端长驻 Agent、无 Slack。参照 **壳层布局与主次关系**；运营类页面收敛到 **设置** 内，避免侧栏膨胀。

### 0.2 与参照范式的对齐关系

| Codex / Workspace 叙事 | KyberKit 实现位置 |
|-------------------------|-------------------|
| 侧栏：新对话、插件、自动化、历史 | **§2 高频侧栏**（一层） |
| 主区：对话流 + 制品预览 | **§3–4** 中栏流 + **§4.3 右栏预览** |
| 治理、合约、审计等低频 | **§5 设置** 子菜单；**工作区概览**见 **§2.3 首次 Modal** |
| 任务进度、文件、工具、下一步 | **§4.2** 运行摘要 + 右栏 **预览 / 运行** Tab 或折叠区 |
| Policy / Permit / 审批 | **设置 → 偏好与合规**；流内 **待审批** 高亮 |

## 1. UX 目标

1. **日常优先**：打开 Console 默认进入 **对话与历史**，侧栏一键触达；减少「找后台页」的心智负担  
2. **制品可审阅**：对话中产生的 Markdown、HTML、CSV 等可在 **同屏预览**，无需切外部编辑器即可完成粗审  
3. **运营不丢**：合约、记忆、技能库、审计、进化、Policy 等 **全部可达**，收纳在 **设置** 子菜单，路径固定、可书签化；**工作区概览**不以设置子页形式存在，见 **§2.3**  
4. **可改稿与命令化（Future）**：支持对话内改稿、命令面板、可拖拽 Splitter、主题切换；作为 `Future` 正文能力定义，不纳入 P1-C 验收

> 不是目标：复刻 Codex 账号体系、插件市场、语音输入；P1-C 不强制实现对话内发送消息（若仅观测，输入区可为占位或跳转 TUI）

## 2. 信息架构（IA）

### 2.1 高频侧栏（唯一常驻导航层）

侧栏 **仅一层列表**（可加图标 + 短标签），自上而下建议顺序：

| 项 | 说明 |
|----|------|
| **新会话** | 创建新 AgentSession 并切到中栏空白流（若后端未接发送，则「新观测会话」占位） |
| **搜索** | 跨会话标题 / 首条用户摘要 / session id 的快捷检索，结果点击跳转中栏 |
| **技能 / 插件** | 打开 **技能面板**：展示当前工作区可用 Skill 列表、路径摘要；勾选/复制指令等与会话输入联动的能力按 `web-console-frontend.md` 细化；**不等同**于设置里的技能库管理 |
| **自动化** | 打开 **自动化摘要面板**（侧栏内嵌或抽屉）：下几次计划运行、最近合约事件；提供「管理合约」→ 跳转 **设置 → 合约** |
| **历史对话** | **项目（工作区）** 可折叠分组 + 其下会话线程（标题、相对时间、状态点）；底部可保留 **独立对话** 区（无项目归属的 session） |

侧栏 **底部固定**：

| 项 | 说明 |
|----|------|
| **设置**（齿轮） | 打开 **设置壳**（见 §5），内含全部子菜单；**不占**侧栏主列表槽位 |

**移除**：侧栏上不再放置「合约 / 进化 / 记忆 / 技能库 / 审计 / 概览」等独立顶级入口。

### 2.3 工作区概览（首次打开 Modal）

**目的**：进入 Console 时一次性建立「工作区健康度、最近事件、增长趋势」的心智，**不占用**侧栏与设置子路由名额。

**触发**：

- 用户首次打开主工作区路由（如 `/c` 或应用默认落地页）时，若 **本地持久化键**（建议 `kyberkit.console.overview.dismissed` + 可选 `workspaceId` 后缀）**未**标记为已关闭，则自动弹出 **Modal**。
- Modal 内内容与 **原 Dashboard** 一致：**KPI 卡片 → 实时事件流（SSE）→ 7 天增长趋势**（布局可纵向滚动；Modal 最大宽度约 `min(960px, 92vw)`，高度约 `min(85vh, …)`，底层主界面可浅遮罩）。

**关闭与不再打扰**：

- 主操作：**开始使用** / **关闭** — 关闭 Modal 并写入上述键，**同 workspace** 下次进入 **不再自动弹出**。
- 可选次要链：**稍后再说** — 仅关闭本次会话 Tab 内不重复弹出（若实现简单，可与「开始使用」共用同一 dismiss 行为）。

**再次查看**（非首次也需可达）：

- 在 **设置壳** 顶部工具区（子导航上方或右侧）提供 **「工作区概览」** 按钮或链接：**仅打开同一 Modal**，不进入 `/settings/...` 独立页。
- 打开时可 **不清除** dismiss 键（即仍为「手动查看」，不恢复首次自动弹出）。

**数据来源**：与旧 Dashboard 相同 — `ContractRegistry.list()`、`WorkspaceGrowthStore.aggregateSince()`、`bus.on('*')`（Modal 打开期间订阅 SSE，关闭时可取消订阅以省连接，由 frontend spec 定）。

### 2.4 路由心智（建议）

- 主工作区：`/c` 或 `/c/:sessionId` — 侧栏 + 中栏 + 右栏；**无** `/settings/overview`  
- 设置工作区：`/settings/*` — 左侧为 **设置子导航**，右侧为各子页内容（与 Codex「点齿轮进深层」一致，也可用全屏模态 + 返回）

### 2.5 跨视图跳转（深链约定）

| 来源入口 | 目标 | 跳转规则 |
|----------|------|----------|
| 审计日志行 | `/c/:sessionId` | 打开对应会话并定位到工具调用锚点（建议 `#tool=<eventId>`） |
| 进化记录卡片 | `/c/:sessionId` | 打开对应 task 的会话，滚动到该任务首次触发的系统消息 |
| 侧栏自动化面板（管理合约） | `/settings/contracts` | 进入合约管理默认 Tab（运行中） |
| 侧栏自动化面板（具体合约） | `/settings/contracts?contractId=<id>` | 打开合约页并高亮目标卡片（或自动展开详情） |
| 侧栏技能/插件面板（管理） | `/settings/skills` | 打开技能库页 |
| 设置壳「工作区概览」 | 当前页面 + Modal | 不变更路由，仅打开概览 Modal |
| 记忆/审计/进化中的来源链接 | `/c/:sessionId` | 打开会话并尽量定位来源片段（无锚点则滚到末尾并高亮 session header） |

> 深链参数与锚点命名最终以 `web-console-frontend.md` 和 `web-console-api.md` 为准。

## 3. 全局布局（Codex 式三栏）

**极简顶栏（可选）**：仅保留产品名 / 当前 workspace 文案（只读或下拉）；**Policy 当前值**不强制放顶栏，以 **设置 → 偏好与合规** 为准，避免与 Codex 顶栏抢视线。

```
┌─ TopBar（极薄，可选）──────────────────────────────────────────────┐
│ KyberKit                              workspace: default ▾        │
├─ Sidebar ─┬─ Stream ──────────────────────┬─ Preview / Run ───────┤
│ 新会话     │ 当前会话标题 ···               │ [预览] [运行] 或单栏切换 │
│ 搜索       │ ─────────────────────────      │                      │
│ 技能/插件  │  User / Assistant 消息流       │  预览：md / web / csv │
│ 自动化     │  内嵌「制成品」卡片可点击       │  或 运行上下文详情    │
│ ─────      │ ─────────────────────────      │                      │
│ 项目 ▾     │  [输入区 · 权限 · 发送]        │                      │
│  · 会话…   │                                │                      │
│ 对话 ▾     │                                │                      │
│  · …       │                                │                      │
│            │                                │                      │
│ [设置]     │                                │                      │
└────────────┴────────────────────────────────┴──────────────────────┘
```

- **Sidebar**：固定宽度 **240–280px**（可容纳「自动化」等四字）；背景浅灰、与主区对比度参考 Codex。  
- **Stream 列**：弹性 `min-w` 保证可读；底部 **悬浮或贴底输入条**（占位文案、附件、`+`、权限、模型/模式若适用、发送）。  
- **Preview / Run 列**：大屏 **≥ 360px** 默认，可拖拽调宽；**窄屏** 右栏改为 **底部 Sheet** 或 **与流切换的 Tab**（`对话` / `预览`）。

### 3.1 右栏：预览与运行上下文的关系

避免三栏再叠多层信息：

- **默认**：右栏展示 **运行上下文**（阶段、下一步、已触达文件、工具摘要、Token/MW），与 Workspace 叙事一致。  
- **用户点击某条消息中的制成品**（见 §4.2）：右栏切换为 **预览**，顶部提供 **返回运行** 或 **Tab：预览 | 运行**。  
- **可选**：无预览且用户折叠运行详情时，右栏可收起，中栏占满。

## 4. 主工作区视图

### 4.1 会话标题栏（Stream 顶）

- 当前会话标题（可编辑若 P2；P1-C 可只读显示 session 摘要）  
- 菜单 `···`：重命名、复制 id、关闭/归档（能力按后端）  
- **不**在标题栏堆叠 Policy 下拉（迁至设置）

### 4.2 对话流（Stream）

- **User**：右侧小圆角气泡（与 Codex 类似）。  
- **Assistant**：左侧宽文本区，支持 Markdown 渲染。  
- **工具调用**：可折叠行，展开见 audit 字段；**待审批** 用黄色条 + 文案重复强调。  
- **制成品（Artifact）**：Assistant 或工具结果中，凡可被识别为 **可预览对象** 的，在流内渲染为 **卡片**（标题、类型图标、`打开预览` / 点击整卡）。  
  - **识别规则（P1-C 建议）**：  
    - 工具 `write_file` / `read_file` 且路径扩展名为 `.md` `.html` `.htm` `.csv`；或  
    - 消息内显式 `artifact://` / 前端约定 JSON 块（由 `web-console-api.md` 与 trajectory 格式约定）。  
  - **点击后**：右栏加载对应预览（§4.3）；若右栏已折叠，先展开再加载。

**运行摘要条（Stream 内、消息上方一条即可）**：一行内显示 **阶段** + **下一步**（与旧版「运行上下文条」字段一致，从 trajectory 推导或由 API 聚合）。

**数据来源**：`TrajectoryRecorder` + `bus.on('agent.*')`；预览内容经 **只读文件 API** 或内嵌于事件负载的片段（大文件走 API）。

### 4.3 制成品预览（右栏）

| 类型 | 呈现方式 |
|------|----------|
| **Markdown** | 安全子集渲染（GFM 可选）；代码块高亮；外链默认不自动加载跟踪像素 |
| **网页（HTML）** | `iframe` **sandbox**；`file://` 或本工作区内路径由后端提供 **同源包装 URL** 或内嵌 HTML 字符串（禁止任意远程 URL 未校验直开，安全细则见 deployment spec） |
| **CSV** | 前端解析为表（行数上限 + 虚拟滚动）；大文件仅预览前 N 行 + 提示下载 |

- 预览栏顶部：**文件名 / 类型**、**在新标签打开**（若适用）、**关闭预览**（回到运行上下文或留空态）。  
- **编辑**：P1-C **不**在预览内联编辑；只读审阅。

### 4.4 侧栏「技能 / 插件」面板

- 列表：Skill 名称、描述一行、路径小字；操作：**插入提示前缀**、**复制路径**、**管理…**（跳转 **设置 → 技能库**）。  
- 与 **设置 → 技能库** 区别：侧栏侧重 **当下会话可用**；设置侧重 **增删改查磁盘文件、审计来源**。

### 4.5 侧栏「自动化」面板

- 展示：下一次 `cron` 触发倒计时（Top 3）、最近 `contract.*` 事件一行列表。  
- CTA：**管理合约** → `/settings/contracts`。

## 5. 设置（统一壳 + 子菜单）

点击侧栏 **设置** 进入。布局：**左侧子导航 + 右侧内容**（桌面）；移动端为 **纵向列表进子页**。

建议子菜单顺序（路由示例）；**不含**独立「工作区概览」路由（概览见 **§2.3 Modal**）。

| 路由段 | 页面 | 说明 |
|--------|------|------|
| `contracts` | **合约** | 原 Contracts |
| `evolution` | **进化记录** | 原 Evolution |
| `memory` | **记忆库** | 原 Memory |
| `skills` | **技能库** | 原 Skills（查看/删除；与侧栏插件面板互补） |
| `audit` | **审计日志** | 原 Audit |
| `preferences` | **偏好与合规** | 原 Settings：Policy Pack、Permit、工作区根路径只读 |

设置壳 **顶栏** 另设 **「工作区概览」** 控件，打开 §2.3 所述 **同一 Modal**（非路由子页）。

各子页 **内容与交互** 与修订前 spec 中合约 / 进化 / 记忆 / 技能 / 审计 / 偏好各页 **保持一致**（卡片、Tab、筛选、二次确认等），仅 **入口** 从侧栏迁至此处；原 **Dashboard 独占页** 取消，由 Modal 承担。

## 6. 设计原则

| 原则 | 含义 |
|------|------|
| **侧栏极简** | 仅 §2.1 高频项 + 底栏设置；禁止在侧栏堆运营类顶级入口 |
| **主区可读** | 流式对话为主；运行状态一行摘要 + 右栏详情，避免中栏堆满表格 |
| **预览安全** | HTML 走 sandbox；文件范围限制在工作区；CSV/HTML 大小上限 |
| **现实可视化** | KPI、审计、轨迹均来自既有数据源，不虚构进度 |
| **只读优先** | 写操作范围不变：合约 pause/revoke、permit 撤销、policy 切换、skill 删除 |
| **实时优先** | 概览 Modal 打开期间 SSE 与主会话流 SSE；关闭 Modal 后可断连概览 SSE（实现可选） |
| **零状态前端** | 切会话/切设置子页重新拉取（策略可在 frontend spec 微调缓存） |
| **改稿让位可读（Future）** | 对话内改稿与命令面板等增强能力不得破坏流式阅读和审计可追溯性 |
| **键盘友好** | `g` `s` 打开设置；`g` `c` 回主会话；子页 `?` 显示快捷键表 |

## 9. 验收标准

1. 侧栏仅含：新会话、搜索、技能/插件、自动化、历史（项目+对话）、底栏设置；**无**合约/进化等独立顶项  
2. **首次**进入主工作区出现 **工作区概览 Modal**（KPI + 事件流 + 7d 趋势），关闭后同 workspace 不再自动弹出；**设置壳顶栏**可再次打开同一 Modal  
3. 设置内 **6 个**子路由均可达（合约、进化、记忆、技能库、审计、偏好与合规），行为与迁出前对应页等价  
4. 主区：**对话流** 可展示工具与 audit；**运行摘要** 至少一行可见  
5. 点击流内制成品卡片后，右栏可对 **.md / .html / .csv** 给出正确预览（或明确降级提示「文件过大」）  
6. 右栏在窄屏有可用的 Sheet/Tab 降级方案  
7. 合约 activate/pause/revoke、Policy 切换、Audit 筛选等验收与修订前一致，仅在设置内完成
