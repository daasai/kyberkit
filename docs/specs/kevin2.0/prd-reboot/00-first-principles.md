## 范围与适用性

本第一性框架的认知循环结构是**通用的**——适用于所有需要在复杂信息中持续产出高责任判断的工作，**不限于某个职业或行业**。

### 锚定场景（说明通用性，非穷举）

| 场景 | 典型工作 | 私有方法核心 |
|---|---|---|
| 产品 / 项目工作 | PRD、规划、复盘 | 优先级哲学、机会评估框架 |
| 自媒体 / 内容创作 | 选题、研究、生产、发布 | 选题嗅觉、内容审美、读者直觉 |
| 个人投资 / 资产管理 | 公司研究、决策、监控 | 估值方法、风险偏好、决策纪律 |
| 研究 / 分析 | 假设、证据、综合 | 研究范式、证据标准 |
| 咨询 / 顾问工作 | 客户工作、方法论应用 | 行业判断、解决方案库 |

每个场景的材料类型、制品类型、连接器、私有方法库都不同，但**认知循环结构相同**。Kevin 的产品定义服务于所有这些场景，不专属于其中任何一个。

### MVP 收敛策略：工程深度收敛 + 产品语言广度

- **工程深做**：MVP 首先在"产品 / 项目工作"场景验证连接器、Artifact 类型、Skill 库的端到端
- **语言通用**：产品叙事、第一次见面体验、差异性标签从 Day 1 起即用领域无关语言，必须支持其他锚定场景的想象空间
- **扩展原则**：换场景应该是"新增连接器和 Skill 库"，而**不是**"产品改造"——这是检验产品定义是否通用的关键标准

> 当后续设计中出现 PM/运营/PRD/周报等具体例子时，它们是**说明性举例**，而非产品定义本身。读者需要在阅读时主动做"换场景测试"——同一个机制能否在自媒体、投资、研究场景里同样成立。

## Q1：目标客户高强度知识工作的本质是什么？

### 核心表述

高强度知识工作的本质，是**在复杂系统中持续完成认知迭代循环**：

```
连接 → 获取/处理 → 洞察 → 认知迭代/决策 → 执行 → 监控
                    ↑           ↑
                （机会/偏差/风险）  （判断/方案/举措）
```

这不是"内容生产流水线"，也不是"任务完成序列"。每一次循环的目标，不只是完成这一次任务，而是让工作者对这个复杂系统的理解更深一层——**认知本身在迭代**。

### 五个深层特征

**① 上下文是稀缺资源，不是时间**

工作的大部分认知成本，不在于"写不出来"，而在于"记不住、找不到、无法确认"。这些上下文分散在文档、数据、消息和记忆里。上下文管理本身就是工作的主体。

**② 私有方法是核心竞争力**

一个 PM 评估功能优先级的框架、一个自媒体作者判断选题的直觉、一个投资者评估公司机会的估值范式、一个分析师解读留存数据的方法论——这些是通过经验沉淀的个人智识资产，无法从通用知识库获得，也无法被替代。这是工作者真正的壁垒。

**③ 工作是周期性重复的，但判断是非标准化的**

PM 每季度写规划，自媒体作者每周做选题策划，投资者每月做持仓复盘——表面重复，但每次的上下文、数据、优先级、判断逻辑都不同。纯模板解决不了问题，因为判断层是活的。

**④ 可信度必须在社会系统中成立**

输出不是在真空中评价的。一份 PRD 要过工程对齐和老板审批，一篇内容要经得住读者的检视和算法的考验，一次投资决策要经得住时间和市场的检验。知识工作的输出质量，最终由它在协作系统或市场系统中被接受的程度来判断。

**⑤ 认知疲劳是真实的损耗**

高频、高责任的判断会耗尽认知资源。周五下午写完第 10 个 PRD 的 PM、凌晨改到第 5 版选题的内容创作者、同时盯三只股票走势的投资者——他们不需要更聪明的工具，他们需要一个**不必从头交代所有上下文就能接着干的工具**。

## Q2：AI 需要解决的高价值问题是什么？

### AI 与人的分工原则

洞察和认知迭代/决策是**人类主权地带**——用户的私有方法、情境判断、责任承担，是不可被替代的。AI 试图"代替"这两步，要么降低用户对自己工作的掌控感，要么因为缺乏私有方法上下文而质量不可控。

正确的分工是：

```
连接 ──→ 获取/处理 ──→  洞察  ──→ 认知迭代/决策 ──→ 执行 ──→ 监控
  ↑                      ↑               ↑                  ↑
AI主导                  前置辅助        后置沉淀           AI主导
（规模、速度、           （准备认知      （捕获判断、        （意图转化、
 连接复杂系统）           环境，让洞察    沉淀私有方法，      过滤信号，
                         更容易发生）    作为下次基础）      持续监控）
```

### AI 的四个高价值介入点

**① 连接与获取/处理（AI 主导）**

人无法处理复杂系统的信息规模、速度和关联复杂度。AI 在这里的价值是明确的：连接数据系统、读取文档材料、处理多来源信息、结构化呈现。这是 AI 的绝对优势区。

**② 洞察的前置辅助（AI 为人服务）**

AI 不替用户形成洞察，而是**准备最好的认知环境**，让用户的洞察更容易、更准确地发生：
- 结构化信息，标注异常和偏差
- 呈现与用户目标相关的数据关联
- 提示可能的盲点（如：PM 场景下"竞品分析提到了 X，但当前方案没有覆盖"；自媒体场景下"你计划做的选题，类似话题上周的留存表现是负面的"；投资场景下"你正在研究的公司，上次决策最关注的现金流指标本季度有显著变化"）
- 减少信息噪声，突出决策关键点

**③ 决策的后置沉淀（AI 为人服务）**

这是最被低估的高价值点。用户的最终判断——接受什么、拒绝什么、为什么这样决策——是私有方法的外显化。AI 在这里的价值是**精确捕获并结构化这些判断**：
- 记录用户接受/拒绝了哪些建议，以及隐含的理由
- 沉淀用户的评估框架和偏好模式
- 成为下一次前置辅助的基础——下次 AI 准备信息时，会按照用户已验证的私有方法来组织

**④ 执行与监控（AI 主导）**

用户意图已经确立，AI 负责将意图转化为数字动作，并持续监控数字世界的变化、过滤与用户目标相关的信号。这里需要 AI 的准确性和持续性，不需要 AI 的判断力。

### 为什么"后置沉淀"是核心竞争力

后置沉淀决定了飞轮能否转起来。如果 AI 只做前端信息处理，每次都是一次性的——用完即走，没有积累。但如果 AI 能精确捕获用户的判断，并把它转化为下一次前置辅助的质量提升，就建立了一个越用越好的系统。

私有方法的学习，是 Kevin 区别于所有通用 AI 工具的最深层护城河。

## Q3：AI Agent 产品竞争的本质是什么？

### 核心命题

AI Agent 产品竞争的本质，是**争夺用户认知循环的最佳协作伙伴位置**。

不是功能竞赛，不是模型竞赛，而是：

> 谁能最好地服务用户的认知迭代循环——最准确地准备认知环境，最精确地学习用户私有方法，最可靠地执行用户意图——谁就赢得了这个用户在这个工作场景上的长期使用。

### 四个竞争维度

**① 记忆的准确性和召回率**

不是"存了多少"，而是**在正确时机召回正确记忆**的能力。错误的记忆比没有记忆更危险——它以确定口吻提供错误上下文，破坏用户信任。

准确性：记忆不幻觉、材料引用不误引、方法应用不错位。
召回率：在用户最需要某个上下文的时刻，这个上下文能够出现。

**② 上下文的真实、全面与匹配能力**

- **真实**：材料来自真实来源，结论有据可查，不补全、不猜测
- **全面**：跨系统、跨时间、跨文档的整合——不只是当前文档的上下文
- **匹配**：在用户认知循环的正确节点，呈现最相关的上下文——而不是把所有信息一次性堆给用户

**③ 私有方法的学习与应用精度**

这是竞争中最难复制的维度。通用 AI 用通用知识生成内容，Kevin 应该用**用户自己验证过的方法论**来准备信息和生成内容。

这不是"记住你喜欢 bullet point 格式"——而是：
- "记住你分析留存问题时惯用同期群分析框架，下次看留存数据时自动按这个框架组织"
- "记住你做选题决策时优先看读者画像匹配度而非热度，下次推荐选题时按这个排序"
- "记住你评估投资机会时先用 DCF 再做敏感性分析，下次研究新标的时按这个流程准备"

**④ 协同网络与技能复利**

个体的上下文积累在团队中形成协同效应：多个团队成员的私有方法可以在 Workspace 层面共享和叠加，形成团队级别的工作智能。

技能复利使每次工作不只是完成任务，而是让下一次同类工作的启动成本更低、质量更高。

### 竞争护城河的本质

上下文在哪里积累，哪里就有护城河。用户在 Kevin 中积累的 Materials、Artifacts、决策记录、私有方法——这些是不可迁移的工作资产。离开 Kevin 意味着放弃这些积累，回到从零开始的状态。

这个护城河不靠锁定，靠**让用户真实感受到积累的价值**。

## Q4：如何在信息爆炸时代最短时间获取用户最大化认同？

### 五项设计原则

**① 主动，非被动**

Kevin 在用户认知循环的关键节点主动出现，而不是等待被召唤。

在洞察前置阶段：主动说"你的数据里有一个异常值，可能跟你上周提到的问题有关"。
在执行后置阶段：主动说"你刚才的决策和上次类似场景不同，要记录这次的理由吗"。

主动介入是有边界的——AI 在自己的主权地带（数据处理、信息准备、执行、监控）主动，不在人类主权地带（判断、决策）主动替代。

**② 非人，但有人设**

Kevin 不应该假装是人，但必须有一个**稳定的、可识别的智识性格**。

建议性格定义：
- **严谨**：不猜测，不补全，不确定时明确说不确定
- **主动**：在用户需要之前出现，而不是被动等待
- **直接**：不过度礼貌，不绕弯子，给出真实判断
- **透明**：解释自己为什么这么认为，说清楚自己的依据
- **尊重用户主权**：最终决策权在用户，Kevin 提供判断但不施压

这个性格在每一次交互中一致表达——从 Evidence Badge 的措辞到 Sign-off 的提示文案，都是同一个 Kevin 在说话。

**③ 价值感和差异：从第一刻持续释放**

冷启动第一次交互不能只是"展示功能"，必须**立刻产生一个在其他产品上无法复现的体验时刻**。

第一刻体验的设计原则（场景无关）：
- Kevin 读懂用户提供的材料
- 主动指出一个用户没注意到、但确实存在的关联或缺口
- 通过提问邀请用户做判断，而非替用户做判断
- 措辞展现"严谨 + 主动 + 直接"的智识性格

具体场景示例（说明性，不穷举）：
- **产品场景**："访谈笔记提到了 X 风险，但当前竞品分析没有对应内容——要不要补一段调研？"
- **自媒体场景**："这周的选题草稿和你上月一篇高互动文章的角度有重叠——要不要换个切入点，或者作为系列后续？"
- **投资场景**："这家公司你上次研究时最关注现金流，最近一份财报里这项数据有明显变化——要不要先看这个？"

这一句话传递的信息：Kevin 读懂了材料、发现了关联、主动提问、不替用户决定。

这个时刻之后，每一次交互都应该继续强化这种感知——**Kevin 越来越懂这个项目，越来越懂这个人的工作方式**。

**④ 把好故事讲好**

用户能用一句话清楚地告诉别人 Kevin 是什么，以及它和其他 AI 的不同。这句话不应该来自产品介绍，而应该来自用户的自发表达——说明产品体验本身就已经是那个故事。

**⑤ 差异性标签（待定，是最重要的未解决问题）**

Kevin 需要一个独一无二的身份标签——不是功能描述，而是体验革新的一句话表达。

分析已知方向：

| 方向 | 候选表达 | 优势 | 风险 |
|---|---|---|---|
| 上下文积累 | "唯一记得你工作的 AI" | 立即可理解 | 可能被追上（ChatGPT Memory） |
| 私有方法放大 | "把你的经验变成 AI 的判断力" | 触及最深护城河 | 用户认知成本高 |
| 认知分工 | "AI 做数据，你做判断" | 清晰的人机关系 | 可能限制想象空间 |
| 掌控感 | "你批准的，才会发生" | 时代情绪契合 | 暗示"麻烦的流程"风险 |
| 认知伙伴 | "让你的判断在数字世界发挥" | 最本质 | 还没有找到简洁表达 |

**寻找标签的正确方法**：先确定第一次见面的最震撼时刻是什么，标签从那个体验里提炼，而不是从功能描述里推演。标签是体验的语言凝结，不是策略的文字表达。

## Q5：如何构建"复利"和"信任"飞轮？

### 飞轮的两个引擎

认知循环分工决定了飞轮有两个相互强化的引擎：

**数据引擎（前置辅助质量的提升）：**

```
更多材料积累
    ↓
信息准备更全面、更精准
    ↓
洞察机会更容易被识别
    ↓
用户形成更好的判断
    ↓
判断结果被记录
    ↓
下次材料积累更有方向
```

**方法引擎（后置沉淀质量的提升）：**

```
用户做出判断（接受/拒绝/修改/补充）
    ↓
Kevin 捕获判断 + 提取方法模式
    ↓
私有方法被结构化（Skill 草案）
    ↓
下次 Kevin 按用户方法准备信息
    ↓
信息准备更符合用户思维框架
    ↓
洞察质量提升，判断更快
    ↓
用户做出更多判断，被捕获
```

两个引擎合并，就是完整的复利飞轮：**每一次工作，Kevin 既积累了数据，也积累了方法，下一次同类工作的启动质量永远高于上一次。**

### 信任飞轮

信任不是靠声称"Kevin 很准确"——是靠让用户**自己发现 Kevin 是准确的**，并在每一次发现中主动加深使用。

```
Kevin 主动提示一个关联（前置辅助）
    ↓
用户核实，发现是真实的
    ↓
"Kevin 发现了我没注意到的东西"（信任 +1）
    ↓
用户将更重要的判断场景交给 Kevin 的信息准备
    ↓
Kevin 有更高质量的上下文学习机会
    ↓
下次 Kevin 的前置辅助质量更高
    ↓
用户核实正确率更高（信任 +N）
```

**信任飞轮的关键设计要求：**

- **主动邀请验证**：Evidence Badge 不只是显示，而是主动引导用户去核实（"这是 Kevin 引用最频繁的片段，点击确认是否正确使用"）
- **诚实对待不确定性**：Kevin 不确定的地方明确标注，用户核实后发现 Kevin 没有隐瞒不确定性，信任反而增加
- **决策记忆展示**：第二次类似任务时，Kevin 主动说"上次你处理类似问题时选择了 X，这次情况类似，是否继续同样的方向？"——让用户感受到记忆是真实且有用的

### 飞轮的可见性设计

飞轮在数据库里转没有意义，必须让用户**感知到每一圈的收益**：

| 飞轮里程碑 | Kevin 的主动表达 |
|---|---|
| 第一个 Evidence Badge 被用户核实正确 | "你刚确认了这个引用是准确的。Kevin 会在后续内容里继续用类似方式标注。" |
| 连续 3 次接受同类 diff 建议 | "Kevin 注意到你更倾向于这种处理方式。要设为这个项目的默认风格吗？" |
| 完成第一个完整认知循环闭环 | "你和 Kevin 完成了从材料到制品到执行的第一个完整循环。这些材料和决策将作为下一次同类工作的基础。" |
| 同类工作第 N 次比第 1 次快 | "这是你第 3 次完成同类工作。这次的信息准备使用了上两次沉淀的偏好，减少了 2 轮修改。" |
| 私有方法形成 Skill | "Kevin 观察到你处理这类问题时惯用某种框架，这个模式已经在 5 次工作中出现。要保存为你的方法吗？[预览 Skill]" |

## 综合：对 Kevin 2.0 设计的三个根本性要求

### 要求一：产品架构对齐认知循环，而非技术对象模型

当前 Kevin 的架构语言是 Workspace → Materials → Artifacts → ActionRequest → Audit。这是技术视角下的对象层次，是正确的，但**不应该是用户的心智模型**。

用户的心智模型应该是：**"我在做一个关于 X 的判断循环，Kevin 在准备数据、记录我的判断、执行我的决策、监控后续变化。"**

产品架构不需要改变，但所有入口、引导、文案、交互，都应该用认知循环的语言来表达，而不是技术对象的语言。

### 要求二：前置辅助和后置沉淀必须是可设计的行为，不是抽象原则

前置辅助：在哪些节点、基于什么条件、用什么语气主动介入——这需要"Kevin 主动行为规格"文档，明确每一类主动介入的触发条件、介入内容和表达方式。

后置沉淀：如何捕获用户判断（diff 决策、手动修改、明确拒绝）、如何从中提取私有方法模式、如何以 Skill 形式让用户审查确认——这是 Skill Forge 的核心设计任务，但需要重新定义 Skill 的粒度：**不是工作流偏好，而是判断框架**。

### 要求三：差异性标签是最优先的未解决设计问题

产品有了正确的架构、正确的分工、正确的飞轮，但如果没有一个让用户在 60 秒内就能感知并记住的差异性表达，产品将在竞争中消失在噪声里。

这个标签不来自策略推演，来自**第一次见面的最震撼时刻**。

设计优先级：先确定哪一个第一次见面时刻是不可替代的，再从那个时刻里提炼标签。当前的候选方向是"AI 的前置辅助让用户形成更好的洞察"和"AI 的后置沉淀让用户的方法越用越强"——但这两个方向还没有被凝结成一句足够简单、足够震撼的话。
