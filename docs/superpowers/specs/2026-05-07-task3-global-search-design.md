# Kevin v1.5 Task 3 — Global Search（中栏弹出式搜索框）设计规格

> 状态：Draft（待评审）
> 关联 UAT：`UAT-20260507-003`（Sidebar 搜索点击无反应）
> 关联计划：`docs/superpowers/plans/2026-05-07-kevin15-uat-improvement-implementation-plan.md` Task 3

## 1. 背景与问题

当前 Sidebar 的“搜索”入口无法形成明确、可操作反馈（用户点击后无独立搜索体验），已在 UAT 中被判定为 `v1.5-defect`（P0）。

本次 Task 3 的目标不是一次性做完整“全域索引系统”，而是先交付一个 **最小可用且语义正确** 的中栏弹出式搜索框，保证：

- 点击搜索后有明确视图切换；
- 用户可以通过关键词检索并打开历史会话；
- 为后续文档库/Sensor 深度检索预留扩展位。

## 2. 目标与非目标

### 2.1 目标（本次必须交付）

1. Sidebar 点击“搜索”后，在中栏上方弹出搜索框（不覆盖/替换中栏主制品内容）。
2. 搜索视图支持关键词过滤 **历史会话**（title + artifactPreview）。
3. 搜索结果可一键切换会话（回到编辑主视图并加载该会话上下文）。
4. 提供清晰空态、加载态（最小）和错误提示（最小）。

### 2.2 非目标（本次不做）

1. 不做向量检索、召回排序、语义搜索。
2. 不做跨文档库全文索引（仅预留文案/结构）。
3. 不改动右栏输入能力（`@`、`/` 等在 Task 8）。
4. 不改动 Space/会话模型（已在 Task 1/2 处理）。

## 3. 信息架构与交互

## 3.1 入口与展示方式

- `AppShell` 维护 `searchOpen` 布尔状态。
- `LeftSidebar.onOpenSearch` 触发 `setSearchOpen(true)`。
- `GlobalSearchView` 以 modal 形式覆盖在中栏之上，关闭后 `setSearchOpen(false)`。

## 3.2 搜索弹框结构（中栏覆盖层）

1. 顶部栏：关闭/返回按钮 + 标题“全局搜索”。
2. 搜索输入框：自动聚焦，输入即筛选。
3. 结果区：
   - 分组 A：历史会话结果（本次唯一真实数据源）；
   - 分组 B：文档库/Sensor（占位说明，标注“后续索引接线”）。
4. 空态：
   - 无关键词时展示空白引导页（说明可搜索范围与使用提示）；
   - 有关键词无结果时展示“未匹配到会话结果”。

## 3.3 结果点击行为

- 点击会话结果：
  1) `setActiveSessionId(id)`；
  2) 关闭搜索弹框；
  3) 由现有会话/制品加载链路接管（不新增旁路逻辑）。

## 4. 数据来源与过滤规则

本次数据仅使用 `useSession().sessions`：

- 匹配字段：`title`、`artifactPreview`；
- 匹配方式：大小写不敏感 `includes`；
- 排序：沿用 `sessions` 现有顺序（通常按 `updatedAt desc`）。

后续扩展点（不在本次实现）：

- 文档库树检索（按 Space）；
- Sensor 元数据检索；
- 混合排序策略（时间 + 相关度）。

## 5. 状态与错误处理

1. 加载态：若 `sessions` 为空且 Sidecar 不可用，显示“暂不可用，请稍后重试”。
2. 错误态：不新增全局错误机制，使用视图内轻提示文案。
3. 结果渲染失败：兜底显示空列表，不阻塞返回编辑视图。
4. 关闭交互：支持 `Esc` 关闭、点击遮罩关闭、点击结果后关闭。

## 6. 视觉与文案原则

- 复用现有 `CenterPanel` 设计 token（颜色、边框、字体尺度）。
- 不引入“预装 Skill”暗示性文案。
- 明确标注“文档库 / Sensor 深度搜索将在后续索引接线后启用”。

## 7. 验收标准（DoD）

1. 点击 Sidebar“搜索”后，中栏主制品区保持不切页，搜索以弹框出现。
2. 输入关键词可筛选会话结果。
3. 点击结果后会话切换成功并关闭搜索弹框。
4. 空结果与无数据场景有可理解提示。
5. 相关测试通过（至少覆盖入口切换与结果点击）。

## 8. 测试建议

1. 组件测试：`GlobalSearchView`
   - 输入过滤行为；
   - 点击结果调用 `setActiveSessionId`。
2. 集成测试：`AppShell` + `LeftSidebar`
   - 点击搜索入口后 `searchOpen=true`，且中栏主视图不切换；
   - 搜索结果点击后关闭弹框。

## 9. 风险与取舍

1. 仅会话检索可能让用户预期落差（误以为搜全库）：
   - 通过显式文案管理预期。
2. 当前 Sidecar 会话仍有 Space 作用域不完整风险：
   - 本次不在 Task 3 内解决，依赖后续 Task 6-A/Sidecar 改造。
